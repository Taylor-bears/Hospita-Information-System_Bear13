# 启动前端与后端并联通说明

## 架构现状
- 后端：`medical-system/backend/main.py`（FastAPI，推荐端口 `8000`）
- 备用后端：`medical-system/backend/server.js`（Express，同端口 `8000`，与 FastAPI 二选一）
- 前端：两套 Vite+React 工程
  - 根目录前端：`package.json`、`vite.config.ts`、`src/main.tsx`
  - 医疗系统前端：`medical-system/frontend` 下同结构
- 联通方式：前端通过 `axios` 直连 `http://localhost:8000`，未配置 Vite 代理

## 启动后端（FastAPI，推荐）
- 进入项目根目录：`cd c:\Users\汪小屿\Desktop\traec\Hospita-Information-System_Bear13`
- 运行：`python -m uvicorn medical-system.backend.main:app --reload --port 8000`
- 访问健康检查：`http://localhost:8000/health`

## 启动前端（二选一）
- 根目录前端：
  - `cd c:\Users\汪小屿\Desktop\traec\Hospita-Information-System_Bear13`
  - `npm run dev`
  - 访问：`http://localhost:5173`
- 医疗系统前端：
  - `cd c:\Users\汪小屿\Desktop\traec\Hospita-Information-System_Bear13\medical-system\frontend`
  - `npm run dev`
  - 访问：`http://localhost:5173`

## 联通与验证
- 确认后端已在 `8000` 端口运行（`/health` 返回 OK）
- 前端打开后，执行任一接口（如我的预约列表）应成功请求 `http://localhost:8000/...`
- 若接口失败，检查前端 `axios` 基础地址：
  - 根前端：`src/lib/api.ts`
  - 医疗系统前端：`medical-system/frontend/src/api/axios.ts`

## 可选改进
- 如需同源开发，可在 `vite.config.ts` 添加 `server.proxy` 指向 `8000`，或将后端改为提供静态构建文件（生产环境）。
- 如果改用 Node 备用后端（`server.js`），确保未与 FastAPI 同时占用 `8000` 端口。
